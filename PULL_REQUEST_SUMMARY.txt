# Pull Request Summary: Fix NoSuchMethodError in MockDeviceFragment

## Overview
This PR fixes a fatal crash that occurs when attempting to open the Mock Device Fragment settings screen.

## Problem
The app crashes immediately with:
```
java.lang.NoSuchMethodError: No virtual method getPreferenceManager()Landroidx/preference/j; 
in class Landroidx/preference/z;
```

## Root Cause
The code in `MockDeviceFragment.onCreatePreferences()` was calling:
- `getPreferenceManager()` expecting return type `j` (DialogInterface listener)
- But the actual return type is `F` (PreferenceManager)
- Then trying to call non-existent `setSharedPreferencesName()` method

This happened due to obfuscation mismatches in the androidx.preference library.

## Solution
Removed the 5 problematic lines (638-645) that attempted to set the SharedPreferences name.
This is safe because:
1. All preference operations already use direct SharedPreferences access
2. Methods use `Context.getSharedPreferences("mock_device_prefs", 0)` directly
3. PreferenceManager name is not actually used anywhere
